================================================================================
IMPORTANT NOTES:
================================================================================

1. VOLTAGE: Most IMU sensors operate at 3.3V. DO NOT connect to 5V pins!
   - Raspberry Pi GPIO is 3.3V tolerant
   - Using 5V can damage both the sensor and Raspberry Pi

2. I2C BUS: Default I2C bus on Raspberry Pi is I2C1 (GPIO 2/3)
   - Enable I2C: sudo raspi-config → Interface Options → I2C → Enable
   - Check devices: i2cdetect -y 1

3. PULL-UP RESISTORS: Most breakout boards include pull-up resistors (typically 10kΩ)
   - If using raw sensor, add 4.7kΩ - 10kΩ pull-ups on SDA and SCL

4. PHYSICAL PINS: Refer to Raspberry Pi pinout diagram
   - Physical pin 1 is near the SD card slot
   - BCM numbering is used in software (GPIO 2, GPIO 3, etc.)

5. MULTIPLE SENSORS: You can connect multiple I2C devices on the same bus
   - Use AD0/ADDR pins to change I2C addresses (if supported)
   - MPU6050: AD0 LOW = 0x68, AD0 HIGH = 0x69
   - Some sensors have fixed addresses (BNO055 = 0x28 or 0x29)

6. I2C ADDRESSES (common):
   - MPU6050/MPU6500: 0x68 (default), 0x69 (if AD0 HIGH)
   - MPU9250: 0x68 (default), 0x69 (if AD0 HIGH)
   - BNO055: 0x28 (default), 0x29 (alternative)
   - LSM6DS3/LSM6DS33: 0x6A (default), 0x6B (alternative)

================================================================================
TROUBLESHOOTING:
================================================================================

- No device detected: Check I2C is enabled (sudo raspi-config)
- Permission denied: Add user to i2c group: sudo usermod -a -G i2c $USER
- Wrong readings: Verify 3.3V connection (not 5V)
- I2C bus error: Check wiring, pull-up resistors, and I2C speed (default 100kHz)
